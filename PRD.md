# 💧 喝水助手 — 产品需求文档

**版本**：v1.0  
**状态**：已实现 ✅

---

## 1. 产品概述

### 1.1 定位
一款轻量级 Web 应用，定时通过 **企业微信 ClawBot 机器人** 向用户发送趣味喝水提醒，并提供手机端管理界面。

### 1.2 目标用户
- **办公族**：久坐电脑前，容易忘记喝水
- **学生**：自习/上课时段需要定时提醒
- **团队**：多成员各自接收提醒，独立管理饮水数据

### 1.3 核心价值
帮助用户养成规律饮水习惯，通过 **定时提醒 + 趣味文案 + 数据追踪** 形成正向循环。

## 2. 技术架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js 16 (App Router) | SSR + API Routes 一体化 |
| 样式方案 | Tailwind CSS 4 | 移动端优先，原子化 CSS |
| 定时任务 | node-cron | 每分钟 tick，整点发送 |
| 消息推送 | @claw-lab/wxclawbot-cli | 企业微信 ClawBot SDK |
| 数据存储 | 本地 JSON 文件 (持久卷) | 轻量无依赖 |
| 部署平台 | Railway | 支持 Node 常驻进程 + 持久化存储 |
| 时区 | 中国时区 (UTC+8) | 所有时间计算基于北京时间 |

## 3. 功能清单

### 3.1 已实现功能

| 功能 | 优先级 | 描述 |
|------|-------|------|
| F1. 扫码登录 | P0 | 通过 ClawBot 二维码登录，自动绑定企业微信 |
| F2. 定时喝水提醒 | P0 | 每天 8:00~22:00 每小时整点推送 |
| F3. 喝水记录 | P0 | 点击「喝了一杯」记录本次饮水量 |
| F4. 今日统计 | P0 | 进度条 + 杯数 + 毫升数 + 百分比 |
| F5. 提醒开关 | P0 | 一键开启/关闭定时提醒 |
| F6. 提醒时段设置 | P1 | 自定义起止时间（默认 8:00~22:00） |
| F7. 每日目标设置 | P1 | 自定义目标杯数（默认 8 杯）和杯容量（默认 250ml） |
| F8. 趣味文案库 | P1 | 24 条随机文案，支持 {days}/{remaining}/{name} 变量 |
| F9. 连续坚持天数 | P1 | 自动统计连续喝水天数 |
| F10. 多用户支持 | P0 | 每人独立登录、独立凭证、独立数据 |
| F11. 用户切换 | P1 | 管理页面顶部切换查看不同用户的数据 |
| F12. 发送测试提醒 | P1 | 手动触发一次提醒，用于验证配置 |
| F13. 自动重试 | P1 | 限频时每 5 分钟重试，最多 12 次（1 小时） |
| F14. 中国时区 | P0 | 所有时间基于 UTC+8 北京时间 |

### 3.2 趣味文案示例

```
俏皮型："💧 水在等你，就像我在等你喝水～"
关心型："🥤 亲爱的，该喝水了！皮肤需要你～"
知识型："🧠 大脑缺水会变迟钝，快喝一杯！"
鼓励型："🌟 你已经坚持 {days} 天了，继续加油！"
紧迫型："🚨 缺水警报！请立即补水！"
```

## 5. 用户流程

### 5.1 新用户首次使用

```
访问首页 → 点击扫码登录 → 微信扫码 → 手机上确认
→ 凭证自动保存 → 跳转首页 → 进入管理页面
→ 点击「喝了一杯」 → 等待整点提醒
```

### 5.2 多用户场景

```
用户A 扫码 → 保存A的凭证 → A收到A的提醒 ✓
用户B 扫码 → 保存B的凭证 → B收到B的提醒 ✓

在管理页面：
  用户A 点击自己的标签 → 看到A的喝水量
  用户B 点击自己的标签 → 看到B的喝水量
```

### 5.3 每日提醒流程

```
08:00 → 调度器检查 → 用户A提醒开启？→ 在时段内？
  → 生成随机文案 → 通过A的ClawBot发送 → A微信收到
  → 继续检查用户B → ...
09:00 → 同上
...
22:00 → 超出时段，跳过
```

## 6. 已知限制

| 限制 | 说明 | 缓解措施 |
|------|------|---------|
| ClawBot API 限频 | 短时间多次发送会被限频 (ret=-2) | 冷却 3 分钟 + 自动重试 1 小时 |
| 本地 JSON 存储 | 不支持高并发，数据量大会变慢 | 单用户场景下完全够用 |
| 无用户密码 | 依赖 ClawBot 扫码登录 | 扫码即身份认证，无需额外密码 |
| 无数据库 | 数据存储在 Railway 持久卷 | 重启不丢失，迁移需手动拷贝 |
